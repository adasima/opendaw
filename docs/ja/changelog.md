# 変更履歴 (Changelog)

> OpenDAWのアップデート履歴を記録します。

## Phase 30-32: アンドゥ・リドゥ、グリッドスナップ、プラグイン連携の基盤統合

### ✨ 新機能 (New Features)
- **サブバス / センド＆リターンのルーティング**: トラックごとの出力先変更や、他のトラックへのセンドルーティング（送信先の追加・送信量の調整）機能が追加され、SvelteUIのトラック詳細パネルから操作できるようになりました。
- **アンドゥ・リドゥ機能の導入**: プロジェクト状態の履歴管理基盤が構築され、クリップの追加や移動などの操作の取り消しとやり直しが可能になりました。
- **グリッドスナップ機能**: タイムラインやピアノロールでのクリップ・MIDIノートの配置時に、指定した解像度（拍や小節）にスナップ（吸着）させる機能が実装されました。
- **プラグインホスティングのフロントエンド統合**: プラグインブラウザからトラックへのVST3 / CLAPプラグインのロード操作がSvelteUI（ブラウザおよびトラック詳細パネル）とTauriバックエンド間で連携されるようになりました。

### 🚀 改善 (Improvements)
- **バックエンドAPIのリファクタリング**: `app.rs` に集中していたTauri Command群が、`project.rs`, `track.rs`, `clip.rs`, `transport.rs`, `plugin.rs` など機能別のモジュールへ分割・整理されました。

## Phase 25-29: Tauri + WASMアーキテクチャの機能統合と強化

### ✨ 新機能 (New Features)
- **プロジェクト保存・読み込み機能の実装**: プロジェクトのBPM、マスターボリューム、追加されたトラック状態を`.aura`形式で保存・復元できるようになりました。
- **MIDIクリップの管理とピアノロール連携**: タイムラインからMIDIクリップを追加・削除・移動できるようになり、ピアノロールの編集内容がTauriバックエンドへ同期されるようになりました。
- **オーディオクリップの管理統合**: SvelteフロントエンドおよびWASM UIから、オーディオクリップを追加・移動・削除できるようになり、Tauriバックエンドとの状態同期ロジックが拡充されました。

### 🚀 改善 (Improvements)
- **Svelte UIからの状態同期の結合**: トランスポート操作（再生、停止等）やトラック管理（音量、追加、削除等）が、Tauriバックエンドの`ProjectState`へ即座に反映されるようになりました。
- **タイムラインと波形・ノート描画の連携**: Tauriから送信されたプロジェクト状態をもとに、WASMキャンバス上でオーディオ波形およびMIDIノートが描画されるようになりました。
- **リアルタイム性の改善**: オーディオスレッドでのロック競合を排除するため、MIDIルーティング等の状態同期をlock-freeなデータ構造へと移行しました。

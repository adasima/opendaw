# MIDI仕様

> OpenDAWにおけるMIDIの処理、サポートされるメッセージ、およびシーケンス構造のリファレンスです。

## 概要
OpenDAWは、外部MIDI入力デバイスからのメッセージ受信、パース、およびピアノロールで利用される内部シーケンスの管理をサポートしています。このドキュメントでは、実装されているMIDI関連の仕様とデータ構造について説明します。

## MIDI入力デバイス
OpenDAWは `midir` クレートを使用してシステム上のMIDIデバイスを検出し、接続します。
- `OpenDAW MIDI Input` という名前で仮想的な入力ポートを構築し、接続を行います。
- 受信したMIDIメッセージは非同期チャンネルを通じて処理されます。

## サポートされるMIDIメッセージ
現在パースして解釈できるMIDIメッセージ（`MidiMessage`）は以下の通りです。これ以外のメッセージは `Unknown` として無視または保持されます。

| メッセージ | 概要 | 取得データ |
|:---|:---|:---|
| Note On | ノートの開始 | チャンネル、ノート番号、ベロシティ |
| Note Off | ノートの終了 | チャンネル、ノート番号、ベロシティ |
| Control Change | コントロールチェンジ | チャンネル、コントロール番号、値 |
| Pitch Bend | ピッチの変更 | チャンネル、値 (14-bit) |

> ※ ベロシティ `0` の `Note On` メッセージは、自動的に `Note Off` としてパースされます。

## MIDIシーケンスデータ構造
ピアノロールなどで編集されるMIDIシーケンスは、内部で `NoteEvent` のリストとして管理されます。各ノートは以下のようなプロパティを持ちます。

| プロパティ | 型 | 説明 |
|:---|:---|:---|
| id | `usize` | ノートの一意の識別子（自動インクリメント） |
| pitch | `u8` | MIDIノート番号 (0〜127) |
| velocity | `u8` | ベロシティ (0〜127) |
| start_beat | `f64` | ノートの開始位置（拍単位） |
| duration_beats | `f64` | ノートの長さ（拍単位） |

シーケンス (`Sequence`) 全体としては、上記の `NoteEvent` をリストとして保持し、ノートの追加、削除、クリアなどの操作を提供します。

## キーボードショートカット
現在、MIDI設定専用のキーボードショートカットはありません。

## 注意事項・制限
> - 現在のバージョンではMIDI出力（外部ハードウェアへの送信）はサポートされていません。
> - システムエクスクルーシブ (SysEx) メッセージは `Unknown` として扱われ、パースされません。

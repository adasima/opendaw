# MCP API リファレンス

> AIエージェントなどからOpenDAWを外部制御するためのModel Context Protocol (MCP) の仕様です。

## 概要
MCP (Model Context Protocol) サーバー機能を利用することで、AIエージェントや外部アプリケーションからOpenDAWのトランスポート（再生・停止）やトラック操作を制御できます。現在、MCPサーバーはTokioランタイム上で非同期に動作し、UIと連携して各種操作を実行します。

## 基本的な使い方
1. OpenDAWを起動すると、自動的にバックグラウンドでMCPサーバーが起動します。
2. 外部クライアントからMCPプロトコル経由でコマンドを送信します。
3. MCPプロトコル経由で送信されたコマンドは、非同期チャンネルを通じてUIに反映されます。

## 詳細設定（サポートされる操作）
現在実装されているハンドラとサポートされている操作の一覧です。

| ハンドラ | 操作 | 概要 | 実装状況 |
|:---|:---|:---|:---|
| Transport | `play` | 再生を開始します。 | UI連携済み |
| Transport | `stop` | 再生を停止します。 | UI連携済み |
| Transport | `toggle_loop` | ループ再生の有効/無効を切り替えます。 | UI連携済み |
| Tracks | `add_track` | 新しいトラックを追加します。 | UI連携済み |
| Tracks | `remove_track` | 指定したIDのトラックを削除します。 | UI連携済み |

## キーボードショートカット
現在、MCPサーバー専用のキーボードショートカットはありません。

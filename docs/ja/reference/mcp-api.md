# MCP API リファレンス

> AIエージェントなどからOpenDAWを外部制御するためのModel Context Protocol (MCP) の仕様です。

## 概要
MCP (Model Context Protocol) サーバー機能を利用することで、AIエージェントや外部アプリケーションからOpenDAWのトランスポート（再生・停止）やトラック操作を制御できます。現在、MCPサーバーはTokioランタイム上で非同期に動作するスケルトンとして実装されています。

## 基本的な使い方
1. OpenDAWを起動すると、自動的にバックグラウンドでMCPサーバーが起動します。
2. 外部クライアントからMCPプロトコル経由でコマンドを送信します。
3. （※現在のバージョンではUIスレッドとの完全な連携は未実装であり、バックエンドでのコマンド受信とログ出力のみが機能します。）

## 詳細設定（サポートされる操作）
現在実装されているハンドラとサポートされている操作の一覧です。

| ハンドラ | 操作 | 概要 | 実装状況 |
|:---|:---|:---|:---|
| Transport | `play` | 再生を開始します。 | バックエンドログ出力のみ |
| Transport | `stop` | 再生を停止します。 | バックエンドログ出力のみ |
| Transport | `toggle_loop` | ループ再生の有効/無効を切り替えます。 | バックエンドログ出力のみ |
| Tracks | `add_track` | 新しいトラックを追加します。 | バックエンドログ出力のみ |
| Tracks | `remove_track` | 指定したIDのトラックを削除します。 | バックエンドログ出力のみ |

## キーボードショートカット
現在、MCPサーバー専用のキーボードショートカットはありません。

## 注意事項・制限
> - 現在のバージョンでは、MCP側のスケルトン実装に留まっています。UIスレッド（`app.rs`）と連携して実際にUIを更新したり音声を制御する機能は未完成です。
> - TCP/UDPポートや認証などの具体的な接続設定は、今後のバージョンで実装予定です。

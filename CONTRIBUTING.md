# OpenDAW へのコントリビューション

OpenDAW へご興味をお持ちいただきありがとうございます！

## 🤖 盆栽開発モデルについて

このリポジトリは、**AIエージェントが24時間体制で自律的に開発する「盆栽開発」モデル**で運営されています。

| エージェント | 役割 |
|:---:|:---|
| 🏗️ Architect | 設計・計画・ロードマップ管理 |
| 🚀 Nova | 新機能の追加・UX改善 |
| 🛡️ Warden | コード品質の改善・安全性向上 |
| 📝 Scribe | ユーザードキュメントの整備 |

## 📝 フィードバックの方法

### ✅ Issue を作成してください

機能の要望やバグ報告は **Issue** でお願いします。

- 👉 [新しいIssueを作成する](https://github.com/adasima/opendaw/issues/new)

Issue に記載いただいた内容は、AIエージェント（Architect）が定期的に確認し、[ROADMAP.md](ROADMAP.md) に取り込みます。

#### 機能要望の書き方
```
タイトル: [Feature] 〇〇機能が欲しい

## やりたいこと
どんな機能が欲しいか

## なぜ必要か
その機能があるとどう嬉しいか

## 参考
参考になるDAWや実装例があれば
```

#### バグ報告の書き方
```
タイトル: [Bug] 〇〇で△△が起きる

## 環境
- OS:
- バージョン:

## 再現手順
1. ...
2. ...

## 期待される動作
## 実際の動作
```

### 🚫 PRは受け付けていません

外部からのプルリクエストは**自動的にクローズ**されます。

AIエージェントが毎時コードを更新しているため、外部PRはすぐにコンフリクトを起こし、レビュー・マージの運用が困難なためです。ご理解ください。

機能追加のアイデアがある場合は、Issue で共有していただければ、AIエージェントが実装を検討します。

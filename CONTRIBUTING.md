# OpenDAW へのコントリビューション

OpenDAW へご興味をお持ちいただきありがとうございます！

## 🤖 盆栽開発モデルについて

このリポジトリは、**AIエージェントが24時間体制で自律的にコードを追加・整理する「盆栽開発」モデル**で運営されています。

現在、以下の4つのAIエージェントが定期的にPRを作成しています:

| エージェント | 役割 | 頻度 |
|:---:|:---|:---:|
| 🏗️ Architect | 設計・計画・ロードマップ管理 | 1日2回 |
| 🚀 Nova | 新機能の追加・UX改善 | 毎時 |
| 🛡️ Warden | コード品質の改善・安全性向上 | 4時間ごと |
| 📝 Scribe | ユーザードキュメントの整備 | 4時間ごと |

これらのPRは自動マージ設定がされています。

## 👤 人間のコントリビューターの方へ

人間のコントリビューションは大歓迎です！ ただし、AIエージェントとの共存のために、以下の点にご注意ください。

### PRを出す前に

1. **まずIssueを作成してください** — 実装予定の内容を事前に共有することで、AIエージェントとの作業の重複を防げます。
2. **ROADMAP.md を確認してください** — 現在AIが取り組んでいるフェーズとタスクが記載されています。
3. **ARCHITECTURE.md を確認してください** — モジュール構造やファイル配置のルールが定義されています。

### PR作成時のお願い

- **PRのタイトルは日本語で** お願いします（プロジェクトの方針です）
- **1つのPRで1つの変更** にしてください（小さなPRが好まれます）
- `cargo test` と `cargo clippy` が通ることを確認してください
- AIエージェントのPRと同じファイルを変更する場合、**タイミングによってはコンフリクトが発生する可能性**があります。その場合はリベースをお願いすることがあります。

### レビュー

人間のPRは自動マージされません。リポジトリオーナーが手動でレビュー・マージします。

## 📐 コーディング規約

- コメント・docコメントは日本語で記述
- public な API には `///` doc コメントを付与
- マジックナンバーは `const` 定数に
- `engine/` 内のオーディオコールバックではヒープアロケーション禁止（詳細は ARCHITECTURE.md を参照）

## 🐛 バグ報告

Issue を作成し、以下を記載してください:
- OS とバージョン
- 再現手順
- 期待される動作と実際の動作
- 可能であればスクリーンショット
